# P3-3 平台运营后台完善 - 执行级 TDD 计划

> 来源：`P3-3-平台运营后台完善.md`。本文将设计级任务展开为可逐步执行、可提交、可验收的 TDD 队列。

## 执行范围

- 后端：操作审计、审计查询、数据字典、系统参数、渠道开关、通知模板管理、平台监控聚合。
- 前端：`admin-web` 门店监控、操作审计、系统配置三页，并将平台总览数据源切到监控 API。
- 权限：`monitor:view`、`audit:view`、`config:view`、`config:manage`，仅平台角色可访问。
- 验收：触发写操作后审计可查；切换渠道后 resolver 即时生效；平台监控可跨租户汇总与下钻。

## 约束补充

- `audit_logs` 是业务审计表：`tenant_id` 可为空，仍启用 RLS + FORCE。平台 bypass 可查全量；常规租户上下文只能查本租户行；`tenant_id is null` 仅平台 bypass 可见。
- `dictionaries`、`dict_items`、`system_configs`、`channel_configs` 是平台配置表：不带 `tenant_id`，不启用 RLS，必须由平台 RBAC 守门。
- 审计落库不得阻塞主业务：拦截器捕获主流程成功或失败后旁路记录；审计失败只记录日志。
- 配置热切换先支持单实例内存缓存 + EventBus 广播；P3-4 再增强多级缓存和熔断。
- real provider 在 P4-4 前只允许注册占位，解析到 real 时必须显式抛 `NOT_IMPLEMENTED`，不能静默降级。
