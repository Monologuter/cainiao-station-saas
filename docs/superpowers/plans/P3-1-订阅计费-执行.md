# P3-1 订阅计费 · 逐步 TDD 执行计划

> 展开日期：2026-06-18  
> 来源：`P3-1-订阅计费.md`、`全周期实现计划总表.md`、`设计方案-整合版.md`。  
> 原则：billing 是独立限界上下文；除平台级 `plans` 外，`subscriptions/invoices/usage_records/usage_dedup` 全部带 `tenant_id` + RLS + `FORCE ROW LEVEL SECURITY`；金额全程以分为单位；出账与计量必须幂等。

## 0. 当前基线与落位约定

- 已有基线：
  - `tenant.status` 已有 `ACTIVE/SUSPENDED/CLOSED`，可承接欠费停用。
  - `pay` 已有 `PayChannel`、`Payment` 表和 `PayService.payShipOrder`，当前只支持 `PaymentBizType.SHIP_ORDER`。
  - `notify` 发送成功会写 `notifications(status=SENT, channel=SMS)`，尚未发布短信发送结果事件。
  - `core/queue` 已有 BullMQ/Redis 基础设施与 repeatable registrar，可作为 P3-1 job 参照。
- 后端落位：
  - `backend/apps/api/src/modules/billing/`，按 `plan/ subscription/ usage/ invoice/ jobs/ pay/` 分层。
  - Prisma 新增 `Plan`、`Subscription`、`UsageRecord`、`UsageDedup`、`Invoice` 及相关枚举。
  - `PaymentBizType` 扩展 `SUBSCRIPTION_INVOICE`，billing 支付走独立 `SubscriptionPayService`，不复用 shipping 状态机。
- 前端落位：
  - `admin-web` 继续扩展套餐配置、订阅与账单页。
  - `station-web` 在设置/订阅视图中提供「我的订阅/账单/用量」。
- 本期新增权限码：
  - 平台/租户：`plan:read`、`plan:write`、`subscription:read`、`subscription:write`、`subscription:admin`、`usage:read`、`usage:meter`、`invoice:read`、`invoice:pay`、`invoice:run`、`invoice:admin`。
