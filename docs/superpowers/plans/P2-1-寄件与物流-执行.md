## 0. 当前基线与落位约定

- 后端实际目录为 `backend/apps/api/src/{core,modules/<context>}`；本期新增模块固定为 `modules/shipping`、`modules/pay`、`modules/logistics`。
- P1 已完成：JWT/RBAC、`TenantPrismaService.withTenant`、RLS、EventBus、Redis、station/parcel/inbound/pickup/notify、station-web 与 user-app 请求层。
- 新增 Prisma 模型统一使用 camelCase 字段 + `@map("snake_case")`，表名 `@@map("ship_orders")` 等。
- 本期新增表：`ship_orders`、`price_rules`、`payments`、`logistics_tracks`。全部带 `tenant_id`，迁移中启用 RLS 与 `FORCE ROW LEVEL SECURITY`。
- 后端 API 统一前缀 `/api/shipping/*`，响应仍为 `{code,message,data}`；staff 端使用 staff JWT，consumer 端使用 P1-3 的 `pickToken`。
- 新增权限码：`shipping:quote`、`shipping:create`、`shipping:read`、`shipping:pay`、`shipping:collect`、`shipping:cancel`。需同步 seed 角色、auth 菜单与 station-web 权限常量。
- Mock 适配层必须通过接口注入：`PAY_CHANNEL`、`LOGISTICS_PROVIDER`。P1-P3 默认 `mock`，不得把微信支付/快递100细节写进业务服务。
