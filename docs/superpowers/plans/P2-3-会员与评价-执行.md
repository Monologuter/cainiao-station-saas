# P2-3 会员与评价 · 逐步 TDD 执行计划

> 展开日期：2026-06-18  
> 来源：`P2-3-会员与评价.md`、`全周期实现计划总表.md`、`设计方案-整合版.md`。  
> 原则：会员/积分为平台级自有账户，不挂租户 RLS；券、评价、投诉为租户级业务表，必须 `tenant_id` + RLS + `FORCE ROW LEVEL SECURITY`。每个 Task 红灯测试 → 最小实现 → 验证 → commit。

## 0. 当前基线与落位约定

- P1/P2-1/P2-2 已完成：Consumer 登录、ParcelPickedUp、ShipOrderPaid、EventBus、Redis、RLS、station-web/user-app 均可复用。
- 后端落位：
  - `backend/apps/api/src/modules/member/` 承载会员档案、积分、券、签到与 consumer 端会员 API。
  - 新增 `backend/apps/api/src/modules/review/` 承载评价、投诉、满意度与门店管理 API。
  - Prisma 仍在 `backend/prisma/schema.prisma`，新增平台级表不加 RLS，租户级表迁移内补 RLS/FORCE。
- 前端落位：
  - `user-app` 增加会员中心、积分、签到、券、评价/投诉页面。
  - `station-web` 增加评价管理、投诉处理、券模板管理与满意度摘要。
- 本期新增权限码：`review:read`、`review:reply`、`review:manage`、`complaint:read`、`complaint:handle`、`coupon:manage`、`coupon:issue`。
- 事件契约若缺 `consumerPhone`，监听器允许通过 `parcelId` / `shipOrderId` 反查手机号，避免回改上游大范围代码。
