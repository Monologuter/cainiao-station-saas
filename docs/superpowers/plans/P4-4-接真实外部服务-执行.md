# P4-4 接真实外部服务 - 执行级 TDD 计划

> 来源：`P4-4-接真实外部服务.md`。本计划把设计级真实集成任务展开为可提交的纵向 TDD 队列；每个 Task 按红灯测试、最小实现、验证、提交推进。CI/dev 默认走 mock 或本地桩，不触真实生产。

## 执行范围

- `backend/`：短信、微信订阅、微信支付、物流 provider 真实实现与工厂开关；回调验签/幂等/降级；集成冒烟。
- `station-web/admin-web/user-app`：原则上不改业务调用方；仅在既有支付/订阅授权参数需要可选字段时做兼容展示。
- `deploy/`：如需，补充 `.env.example` 与 Docker secret/证书挂载说明。
- 验收：`backend/test/real-integrations.e2e-spec.ts` 覆盖 mock↔real 切换、打桩真路径、失败回降级。

## 约束补充

- 新增真实 provider，不删除 mock；业务服务只依赖既有接口或向后兼容可选字段。
- provider 选择优先 DB 渠道开关，兜底环境变量；密钥缺失时安全回落 mock 并可观测。
- 所有真实外呼必须脱敏日志、超时、错误分类、熔断/降级；测试打桩 SDK/HTTP，不打生产。
- 支付/物流回调必须验签与幂等，金额/订单号以系统记录为权威。
