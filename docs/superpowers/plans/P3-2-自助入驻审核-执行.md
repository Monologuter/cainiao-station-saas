# P3-2 自助入驻审核 · 逐步 TDD 执行计划

> 展开日期：2026-06-18  
> 来源：`P3-2-自助入驻审核.md`、`全周期实现计划总表.md`、`设计方案-整合版.md`、`UI设计规范.md`、`design/mockups/hifi/admin-web/applications.html`。  
> 原则：`tenant_applications` 是平台级申请表，申请时尚无租户，显式不加 `tenant_id` 与租户 RLS；公开端点只允许提交/本人查询，平台端点必须走权限与 bypass 通道；审核通过必须复用 `TenantService.createTenant` 并接 P3-1 订阅。

## 0. 当前基线与落位约定

- 已有基线：
  - `TenantService.createTenant` 可事务内创建租户、默认门店、店长角色、店长用户与默认寄件价格规则。
  - `BillingModule` 已提供套餐、订阅、账单与支付；`SubscriptionService.subscribe` 当前入参为 `planId`。
  - `NotifyService` 当前只有包裹入库/滞留模板发送，P3-2 新增入驻通过/驳回通知入口与事件订阅。
  - 尚无 file 上下文实现；本期按适配层原则新增 mock presign storage，不接真实 MinIO SDK。
- 后端落位：
  - `backend/apps/api/src/modules/tenant/application/`：申请状态机、服务、controller、DTO。
  - `backend/apps/api/src/modules/tenant/onboarding/`：审核通过开通编排。
  - `backend/apps/api/src/modules/tenant/events/tenant.events.ts`：`TenantApproved` / `ApplicationRejected` 事件契约；沿用 `TenantStatusChanged` 事件名。
  - `backend/apps/api/src/modules/file/`：mock `FileStorageService`，提供 upload/download presigned URL。
- 前端落位：
  - 公开申请页放在 `station-web/src/views/OnboardingApplyView.vue`，作为无需登录的公开 H5/官网入口。
  - `admin-web` 新增 `ApplicationsView.vue`，侧栏「入驻审核」可进入列表与审核弹窗。
- 本期新增权限码：
  - `tenant:read` 已存在；新增 `tenant:review`，平台超管默认拥有。
