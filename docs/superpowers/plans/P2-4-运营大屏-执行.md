# P2-4 运营大屏 · 逐步 TDD 执行计划

> 展开日期：2026-06-18  
> 来源：`P2-4-运营大屏.md`、`全周期实现计划总表.md`、`设计方案-整合版.md`。  
> 原则：analytics 只读消费其它上下文事件与明细表；自有表 `metric_daily`、`report_jobs` 必须 `tenant_id` + RLS + `FORCE ROW LEVEL SECURITY`；Redis 只作热层，DB/明细为真值。

## 0. 当前基线与落位约定

- 已有基线：
  - `backend/apps/api/src/modules/analytics/analytics.service.ts` 提供明细直查版 `/analytics/overview`。
  - `backend/test/analytics.e2e-spec.ts` 覆盖租户隔离 overview。
  - `station-web/src/api/analytics.ts` 与经营统计路由占位已存在。
- 后端落位：
  - `backend/apps/api/src/modules/analytics/` 下拆分 `metrics/query/ranking/reconcile/report/realtime/listeners`。
  - Prisma 新增 `MetricDaily`、`ReportJob`；迁移内补 RLS/FORCE。
  - Redis key 统一前缀 `an:`，事件幂等用 `an:dedup:{date}`。
- 前端落位：
  - `station-web/src/views/StatisticsView.vue` 替换占位经营统计页，沿用 kit token + Element Plus + CSS 图表骨架。
  - `admin-web` 当前未初始化完整应用，本期先落 `admin-web/` 最小 Vite/Vue 平台总览雏形与 API 测试。
- 本期新增权限码：`analytics:read`、`analytics:export`、`analytics:reconcile`、`analytics:platform:read`。
