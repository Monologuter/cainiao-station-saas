# P4-3 智能库位与包裹量预测 - 执行级 TDD 计划

> 来源：`P4-3-智能库位与包裹量预测.md`。本计划把设计级智能库位/预测任务展开为可提交的纵向 TDD 队列；每个 Task 按红灯测试、最小实现、验证、提交推进。

## 执行范围

- `ai-service/`：`/slot/recommend` 加权评分、`/forecast/volume` 移动平均/季节性/冷启动兜底。
- `backend/`：库位推荐客户端与 `slot-allocator` 接入、`slot_heat_daily`、`volume_forecasts`、热力/预测查询与重算 API、定时任务与降级。
- `station-web/`：货架热力、推荐来源提示、预测曲线与时段高峰看板。
- 验收：`backend/test/smart-slot-forecast.e2e-spec.ts`、ai-service slot/forecast API 测试、station-web 测试与构建。

## 约束补充

- AI 服务只做无状态建议，不查业务库；后端传候选位/聚合特征，最终占位仍由 station 权威执行。
- 入库不得被 AI 阻塞：ai-service 不可用、超时、空推荐、推荐位被占或灰度关闭时必须回退 P1-2 规则分配。
- 新表 `slot_heat_daily`、`volume_forecasts` 必须含 `tenant_id`、RLS Policy、`FORCE ROW LEVEL SECURITY`。
- 预测失败必须本地移动平均兜底并落库；看板始终有可解释曲线。
- station-web 对齐现有全屏业务布局和设计 token，不引入新图表库之外的重型依赖；优先复用既有 ECharts 接入。

## Task 4：包裹量预测服务与落库

### RED
- ai-service 测试覆盖：
  - `/forecast/volume` 历史 >= 7 天用 MA，>= 28 天可选季节性方法，空历史用 `FALLBACK_MEAN`。
  - DAY/HOUR 粒度均返回 horizon 内预测、上下界与 method。
- 后端 e2e 覆盖：
  - `volume_forecasts` 含 RLS + FORCE。
  - `POST /api/analytics/forecast/run` 从历史指标生成未来 7 天预测并落库。
  - ai-service 失败时本地移动平均兜底仍落库。
  - actual 回填后可计算 MAPE。

### GREEN
- ai-service 新增 `app/forecast/` 与 `/forecast/volume`。
- 后端新增 `forecast.client.ts`、`forecast.service.ts`、`forecast.processor.ts`。
- Prisma model/migration：`VolumeForecast` + enum `ForecastGranularity/ForecastMethod`。
- 新增 `GET /api/analytics/forecast/volume` 与 `POST /api/analytics/forecast/run`。

### 验收
- `ai-service` forecast tests、`backend/test/forecast.e2e-spec.ts` 绿。
- Commit：`feat(analytics): add volume forecasting`
