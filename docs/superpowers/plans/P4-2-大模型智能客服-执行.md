# P4-2 大模型智能客服 - 执行级 TDD 计划

> 来源：`P4-2-大模型智能客服.md`。本计划把设计级客服助手任务展开为可提交的纵向 TDD 队列；每个 Task 按红灯测试、最小实现、验证、提交推进。

## 执行范围

- `backend/`：FAQ 知识库模型、AI 会话/消息模型、RLS、降级 FAQ 助手、ai-service client、受控工具、对前端 SSE/历史 API。
- `ai-service/`：assistant 路由、KnowledgeBase 抽象、Mock/Real LLM provider、RAG 编排、工具调用事件与降级语义。
- `user-app/`：在线客服入口与会话页，支持流式回答、引用、工具过程提示、降级徽标、快捷问题。
- 验收：`backend/test/assistant.e2e-spec.ts`、ai-service assistant API 测试、user-app 测试与 H5 构建。

## 约束补充

- 前端永不直连 `ai-service`；业务身份与工具执行只在 NestJS `ai` 上下文收口。
- `ai-service` 不连接业务数据库；需要业务数据时只发 `tool_call`，由 NestJS 工具执行并返回脱敏 `tool_result`。
- 新业务表 `faq_entries`、`ai_conversations`、`ai_messages` 必须有 RLS；FAQ 平台级条目允许 `tenant_id IS NULL`。
- 大模型不可用、超时、熔断或 `AI_ASSISTANT_MODE=mock` 时必须降级 FAQ，且 `degraded=true`。
- 工具强制本人/租户作用域，不接受模型或前端传入的 `tenantId/consumerId/phone` 等敏感身份字段。
- 真实 Claude provider 本期只做可配置适配层与打桩测试，不提交密钥；默认 mock 可全链路运行。
