# P1-3 前端接入 · 逐步 TDD 执行计划

> 来源：`P1-3-前端接入.md`、`UI设计规范.md`、`design/_kit/kit.css`、`design/mockups/hifi/*`。
> 原则：先补齐前端硬依赖契约，再做 station-web，再做 user-app，最后三端联调。每个 Task 红灯测试/可点通失败先行，绿灯后提交。

## 0. 执行约定

- 后端继续使用 `backend/`，Nest 模块按 `modules/<context>` 落地，不新增顶层别名。
- 前端新增 `station-web/`、`user-app/`，目录按设计文档固定分层；`admin-web/` 本期不实现。
- 权限码以后端 P1-1/P1-2 已落地值为准：`parcel:inbound`、`parcel:pickup`、`parcel:read`、`station:manage`、`station:read`、`tenant:create`、`tenant:read`。前端仅做展示映射，不引入与后端不一致的安全判断。
- 主题 token 以 `design/_kit/kit.css` 与 `design/mockups/theme-switch-demo.html` 为准，station-web 支持 `blue|dark|mint`，user-app 支持浅/暗。
- 所有 API 统一解包 `{code,message,data}`，前端不传租户头，多租户上下文只来自 staff JWT；消费者通道使用独立 `pickToken`。

## Task 0.1 · Auth 菜单与权限契约

- 红灯：新增 auth e2e/单测，断言登录后 `GET /api/auth/permissions` 返回当前用户权限码，`GET /api/auth/menus` 只返回命中权限的菜单；`/auth/me` 至少保留用户基本信息并可带 `perms`。
- 实现：在 `identity` 模块补 service 方法与 controller 端点；菜单常量按 P1 页面分组输出，权限码使用现有后端码。
- 绿灯：`npm run test:e2e -- --runInBand` 中新增用例通过；无权限用户不会看到受限菜单。
- 提交：`feat(backend): expose auth menus and permissions`

## Task 0.2 · Analytics 基础统计契约

- 红灯：新增 analytics e2e，先造入库/取件数据，断言 `GET /api/analytics/overview` 返回 `{inboundToday,pickedToday,inStock,pickupRate,overdueCount,notifyToday}` 且跨租户隔离。
- 实现：新增 `modules/analytics`，只通过本上下文可拥有的数据与领域服务边界读取；使用 `TenantPrismaService.withTenant` 继承 RLS。
- 绿灯：统计值与测试数据一致，另一个租户不可见。
- 提交：`feat(backend): add tenant analytics overview`

## Task 0.3 · Consumer 验证码与只读查件通道

- 红灯：新增 consumer e2e，断言 `send-code` 降级返回 ok，固定验证码验证后拿 `pickToken`；使用 token 能按手机号跨门店查到包裹，不能按任意 id 越权读取其他手机号包裹。
- 实现：新增 `modules/member` 或 `modules/parcel` 下 consumer controller/service；pickToken 与 staff JWT 隔离，scope 仅为消费者只读；查询只返回手机号匹配的必要字段。
- 绿灯：本人手机号可查在库/已取历史，其他手机号包裹返回 404/403。
- 提交：`feat(backend): add consumer parcel lookup`

## Task 0.4 · Parcel 列表/详情契约补齐

- 红灯：新增 parcel controller e2e，断言 `GET /api/parcels` 支持 `status/phoneTail/pickupCode/slot/page/size`，`GET /api/parcels/:id` 返回事件时间线。
- 实现：在 `parcel` 模块补只读 controller/service，使用 RLS 与 `parcel:read` 权限；返回结构 `{list,total,page,size}`。
- 绿灯：station-web 列表所需字段齐全，跨租户不可见。
- 提交：`feat(backend): expose parcel queries`

## Task 1 · station-web 脚手架与主题壳

- 绿灯：`npm test`、`npm run build` 通过；本地页面能切三主题且刷新记忆。
