## 0. 当前基线与落位约定

- 后端实际目录遵循设计基线：`backend/apps/api/src/{core,modules/<context>}`，不是旧示例里的 `backend/src`。
- P1-1 已完成：`PrismaService`、`TenantPrismaService.withTenant`、`TenantContext`、统一响应/异常、JWT/RBAC、开店、RLS e2e。
- 数据库列必须使用 snake_case：Prisma 字段用 `tenantId/stationId`，数据库列用 `@map("tenant_id")/@map("station_id")`；新增表用 `@@map("shelves")` 等 plural snake 表名。
- 每张租户业务表必须有 `tenant_id`、RLS policy、`FORCE ROW LEVEL SECURITY`。RLS SQL 使用：
  `current_setting('app.bypass_rls', true) = 'on' OR tenant_id = NULLIF(current_setting('app.tenant_id', true), '')::uuid`
- 新业务模块落位：
  `modules/station`、`modules/parcel`、`modules/inbound`、`modules/pickup`、`modules/notify`。
- Core 扩展落位：
  `core/event-bus`、`core/redis`、`core/queue`。
- 新增业务码统一放入 `core/http/api-code.ts`，至少包括：
  `LOCK_BUSY`、`NO_FREE_SLOT`、`ILLEGAL_TRANSITION`、`PICKUP_CODE_CONFLICT`、`PARCEL_NOT_FOUND`、`AMBIGUOUS_PICKUP`、`ALREADY_PICKED_UP`、`IDEMPOTENCY_CONFLICT`。
